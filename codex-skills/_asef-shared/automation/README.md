# Automation Contracts

Эта директория описывает формальные контракты перехода между этапами методологии.

## Назначение

Automation contracts нужны для того, чтобы:

- этапы можно было исполнять как независимые skills или агенты
- handoff между этапами был машиночитаемым и предсказуемым
- gate-решения не зависели от неявной интерпретации
- AI и человек работали по общему протоколу передачи состояния

## Состав

- `stage-automation-contracts.md` — единый контракт исполнения и handoff между Stage 0-7
- `scoring-confidence-model.md` — единая модель оценки полноты, согласованности, готовности и риска

## Базовый принцип

Каждый этап обязан:

1. Принять стандартизированный входной пакет.
2. Сформировать стандартизированный выходной пакет.
3. Вернуть gate-решение в одном из допустимых статусов.
4. Явно разделить:
   - факты
   - допущения
   - gaps
   - blocking issues
   - human decisions
