# Demo Input

## Формат

Сырой текст

## Содержимое

Нам нужен сервис для малого e-commerce бизнеса, который помогает автоматически разбирать входящие сообщения от клиентов из нескольких каналов и превращать их в структурированные заявки для команды поддержки.

Сейчас сообщения приходят в Telegram, email и иногда через сайт. Операторы вручную копируют их в таблицу, теряют часть контекста, дублируют обращения и долго понимают, что именно хочет клиент.

Хотим систему, которая:

- собирает сообщения из нескольких каналов
- определяет клиента, если это возможно
- понимает тип обращения
- создает структурированную карточку обращения
- помогает оператору не терять контекст переписки

Нужно, чтобы решение было пригодно для роста, не ломалось при сбоях внешних каналов и позволяло дальше подключать новые источники.

Пока не решено:

- нужен ли автоответ клиенту
- нужно ли сразу строить полноценную аналитику
- нужно ли синхронизироваться с CRM
